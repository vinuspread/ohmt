## Open Graph 설정 기준

모든 페이지에 다음 항목을 포함한다:

```
og:title       → 페이지 title과 동일
og:description → 페이지 description과 동일 (또는 요약)
og:url         → canonical URL과 동일
og:site_name   → "OHMT"
og:type        → "website"
og:image       → 템플릿: /templates/{slug}/og-image.jpg (1200×630)
                 메인/contact: /og-image.png (1200×630)
twitter:card   → "summary_large_image"
```

## 구조화 데이터 (JSON-LD)

### 루트 layout.tsx (전체 적용)
- `Organization` — 사이트 조직 정보
- `WebSite` — 사이트 전체 정보

### 향후 추가 검토
- 템플릿 상세: `Service` 또는 `CreativeWork`
- Contact 페이지: `ContactPage`
- 랜딩 페이지: `BreadcrumbList`

> 실제 없는 가격, 평점, 리뷰는 절대 포함하지 않는다.

---

## 다국어 처리

- `/en`과 `/ko`는 각각 `canonical`이 자기 자신을 가리킨다
- `alternates.languages`로 상호 hreflang 연결
- `x-default`는 `/en`으로 설정
- 루트 `/`는 proxy.ts에서 Accept-Language/IP 기반으로 `/en` 또는 `/ko`로 리다이렉트

---

## 이미지 SEO 기준

- 템플릿 썸네일 `alt`: `"{Template Name} website template preview"`
- OG 이미지 `alt`: `"{Page Title}"` 또는 `"OHMT — {description}"`
- 장식용 이미지: `alt=""`
- 주요 콘텐츠 이미지: 용도를 설명하는 alt 텍스트 필수

## 남은 SEO 개선 과제

- [ ] 랜딩 페이지(`/en`, `/ko`) H1 태그 검토 — 현재 마케팅 문구가 H1으로 명확히 지정되어 있는지 확인
- [ ] 템플릿 상세 페이지 H1 태그 — 각 템플릿 내부 Hero 섹션 제목이 H1인지 확인
- [ ] Contact 페이지 JSON-LD `ContactPage` 타입 추가
- [ ] 랜딩 페이지 FAQ 섹션 있으면 `FAQPage` 구조화 데이터 추가
- [ ] 이미지 alt 텍스트 전수 점검 (특히 랜딩 페이지 썸네일)
- [ ] Google Search Console 색인 요청 (Submit to Google)
- [ ] Core Web Vitals 측정 및 LCP/CLS 최적화
